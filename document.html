REST is an architectural style that describes how various components interact in a distributed hypermedia system

RESTful web APIs typically use four of the HTTP request verbs to implement CRUD operations: post, get, put and delete

All operations are idempotent, only put and delete are not safe.

Create

By create we mean that we add data for a new product item (into the database)

POST api/phones

- To create a new phone and save it to the database, you can do a request to the route: api/phones
- The post request is done to the route api/phones and it requests that the web server accepts the data in the body of the request message
- To ceate a new phone, we need the following data in the form of Json: brand, model, os, screensize and image. Keep in mind that screensize is a number!
- Example data to send as input: { "brand": "Samsung", "model": "galaxy 3", "os": "android", "screensize": 4, "image": "www.google.com/random-image"}
- As a response, you get an object with the message succes or the the error if one occured.
- When creating a new row in a table with the form we send the data in json format, if the table row is created succesfully status code 201 is sent out.

Responses:
Example output: {message: "Sucesfully added the phone!"}
Code Success
- Code 201 Created: we have created the new item to the database, in return we could get a success message

Code Error
- Code 400: we have made a bad request, for example, we forgot to add the screensize or the screensize is not a number
- Code 500: something went wrong on the back-end

Content format: application/json

Retrieve

By retrieve we mean that we list the data of a specific item (from the database)

GET api/phones
GET api/phones/:id

- To get data from the server, you can do a request to the uri's above. The first one returns all the phones available as an array. The second one will give you ONE specific item as an object.
- Example data to retrieve as output (from one specific item): { "brand": "Samsung", "model": "galaxy 3", "os": "android", "screensize": 4, "image": "www.google.com/random-image"}
- In the second route, you can see that the path contains :id. This means that you need to give an id in the parameters. An example would be the id 2. To get a phone from the database with id 2, you need to do the request to '/api/phones/2'
- To show the newly created table row we sent a GET request in json. If the data for the row is succesfully retrieved status code 200 is send out.

Response
Example output: [{example phone object}, {example phone object 2}] => object as in all the fields like model and os
Code Success
- Code 200 OK: we succesfully retrieved the items(s)

Code Error
- Code 404 Not Found: there is no existing item with the requested id
- Code 500: something went wrong on the back-end

Content format: application/json

Update

By update we mean that we change the data of a specific item that already exists (from the database)

PUT api/phones/:id

- The put request is done to the route api/phones/:id and it requests that the server web accepts our new data in the body of the request and puts this new data instead of the old data (into the database)
- To update the phone with for example id=2 as input you need to do your request to /api/phones/2
- To update the phone, just give the new values as an input object. For example: { "brand": "Apple", "model": "Iphone X", "os": "IOS", "screensize": 4, "image": "www.google.com/random-image"} -> the one we created above is now updated
- To show the new updated row we sent a put request in json. The body contains the message. If the data for the row is succesfully updated status code 204 is sent out.
- The put call must be used with caution since it's not safe.

Responses
Example output: {message: "Succesfully updated!"}
Code success
- Code 204 No Content: succesfully updated the item

Code error
- Code 400: we have made a bad request, for example, the screensize is not a number
- Code 404 Not Found: there is no existing item with the requested id
- Code 500: something went wrong on the back-end

Content format: application/json

Delete

By delete we delete a specific item (from the database)

DELETE /api/phones/reset
DELETE /api/phones/:id
- The delete request is done to the route api/phones/:id and it requests that the server web deletes a certain item from (from the database)
- To delete the phone with for example id=2 as input you need to do your request to /api/phones/2
- The delete call must be used with caution since it's not safe. Especially to /reset

Responses
Example output: {message: "Sucesfully deleted the phone with id 2!"}
Code Success
- Code 204 No Content: succesfully deleted the item

Code Error
- Code 400: error occured
- Code 404 Not Found: there is no existing item with the requested id
- Code 500: something went wrong on the back-end

Content format: application/json
- We could get back a message confirming we succesfully deleted the item in json
